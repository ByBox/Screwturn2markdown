
##  Scenario 
* Lo scopo di queste note è puntualizzare alcune novità introdotte con la versione 2009.1 dei componenti forniti dalle varie suites DevExpress

##  Novità 

###  ASPxGridView and Editors Suite 
* **Masked Input**: funzionalità aggiunte a text e date editor; implementato un Asp.net Masked Engine; tastiera e mouse wheel possono essere utilizzate per modificare porzioni dell'edit value [Masked Input Demo](http://demos.devexpress.com/ASPxGridViewDemos/Editors/MaskedInput.aspx)
* **Virtual Paging**: oltre al built-in pager può essere utilizzata la scroll bar verticale; la pagina corrispondente verrà recuperata solo all'atto dello scrolling (la funzionalità di scrolling verticale semplice era già presente)
* **Horizontal Scrolling and Excel Style Column Freezing**: permette di ancorare alcune colonne e scorrere le altre
* **Progress Bar Control**: nuovo controllo che può essere sia stand-alone che all'interno di colonne della grid
* **Multi-Column Mode** in List Box and Combo Box Controls: [Multi-Column Demo](http://demos.devexpress.com/ASPxGridViewDemos/Editors/ComboBoxMultiColumn.aspx)
* **Composite key fields** sono ora supportate
* **Accessibility Compliance**: [Accessibility Demo](http://demos.devexpress.com/ASPxGridViewDemos/Accessibility/Compliance.aspx)

###  WPF 
* Rilascio in **beta** di una serie di controlli basati sulla architettura **Windows Presentation Foundation** (windows o web): Toolbar, Menu, Docking System, Grid, Carousel, Charts 
* L'innovazione principale di WPF è la rimozione di ogni legame con il modello di sviluppo tradizionale di Windows, introdotto con la versione 1.0 del sistema operativo. WPF è basato su un sistema di grafica vettoriale. Un elemento caratteristico è la netta separazione separazione tra interfaccia e funzionalità e all'utilizzo di un linguaggio che meglio del codice possa descrivere interfacce grafiche: il linguaggio XAML(Xaml).
* Deve esserer presente il Framework .NET (3.5) sulla macchina client

###  Silverlight 
* Rilascio in **beta** di una serie di controlli basati sulla architettura del plug-in **Silverlight**: Layout, RichEdit, DataGrid, Menu
* Silverlight, in modo analogo ad altre architetture RIA (come Flash, Java fx, ...), si pone l'obbiettivo di visualizzare all'interno di un browser applicazioni multimediali ad alta interattività; Visual Studio 2008 rende disponibile una serie di risorse e di tool per utilizzare l'ambiente silverlight e dotare di tali funzionalità tradizionali progetti ASP.NET 
* In comune con WPF c'è l'utilizzo dello Xaml per il disegno delle intefaccie grafiche
* Browser Supportati: Internet Explorer, Firefox, Safari, ...
* OS supportati: windows e Mac da Microsoft, Linux da Mono
